# EssentiaX v1.0.9 Release Notes

## üêõ Bug Fixes

### Fixed SmartViz Import Error
- **Issue**: Demo files were importing `SmartViz` class which doesn't exist in the public API
- **Fix**: Updated all demo files to use the correct `smart_viz` function
- **Impact**: Colab demos and LinkedIn demos now work without ImportError

### Files Updated
- `COLAB_DEMO.py` - Fixed visualization import
- `COLAB_DEMO.md` - Updated to use correct function
- `linkedin_demo_complete.py` - Fixed import and usage
- `linkedin_demo_visual.py` - Removed incorrect import
- `setup.py` - Bumped version to 1.0.9
- `essentiax/__init__.py` - Added `__version__` attribute

## üìù Correct Usage

### Before (Incorrect)
```python
from essentiax.visuals import SmartViz

viz = SmartViz()
viz.plot_all(df)
```

### After (Correct)
```python
from essentiax.visuals import smart_viz

smart_viz(df)  # Auto mode
# or
smart_viz(df, mode='manual', columns=['col1', 'col2'])
```
